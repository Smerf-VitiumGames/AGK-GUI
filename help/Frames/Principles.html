Defaults


  Creating custom gadgets is a superior way to manage your programs but its is also much more involved.
  There are 4 main ways to set up a program.   Templates, source code and gadgets and includes,   Gui's tend to get complex
  and diffcicult to manage very quicky with multiple line for color size position ect.
  So I reccomend building a menu in an include file and running it in the templates file.

Principles.
Defaults.
  Each type of gadget has default vales located in the Defaults file. It is highly recomended to set the default values that
  you would like each type of gadget to start with. These do not set them in stone each gadget can be altered at any time, but
  this saves time on color positioning setting  images and padding ect.
  For example, A if the defaut color for a Window is set to blue all Windows will be set with the color blue when created.
  Defaults are not part of the api language because it is much quicker to edit each type indevidually. The #include .
  File named "Default can be edited manually.
